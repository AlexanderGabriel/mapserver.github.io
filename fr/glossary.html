Glossaire¶

AGG
Anti-Grain Geometry est un moteur de rendu graphique de haute qualité que peut utiliser MapServer 5.0+. Il gère l’anticrênelage en dessous du pixel ainsi que d’autres fonctionnalités.
CGI
Wikipedia propose une très bonne définition sur CGI.
EPSG

Les codes EPSG sont des codes numériques associés à des définitions de système de coordonnées. Par exemple, le code EPSG:4326 correspond au WGS84 géographique, et EPSG:32611 correspond au “UTM zone 11 Nord, WGS84”. Le protocole WMS utilise les codes EPSG pour décrire les systèmes de coordonnées. Les codes EPSG sont publiés par le comité de Positionnement et de Recherche de l’OGP. Une liste de définitions PROJ.4 correspondant aux codes EPSG peut être trouvée dans le fichier /usr/local/share/proj/epsg. PROJECTION décrit comment les utiliser dans votre Mapfile.

Voir aussi

http://spatialreference.org for a listing of spatial references and an interface to search for spatial references.

Filter Encoding

Filter Encoding est un standard de l’OGC qui définie un encodage XML pour des expressions de filtre pour permettre des requêtes attributaires et spatiales.

Voir aussi

Filter Encoding WFS

FreeType
FreeType est un moteur de police que MapServer utilise pour accéder et afficher des polices TrueType.
GD
GD est une bibliothèque graphique pour la génération dynamique d’images. Il a été le premier moteur d’affichage graphique qui était disponible pour MapServer, et il est nécessaire pour MapServer puisse fonctionner.
GDAL
GDAL (Geospatial Data Abstraction Library) est une bibliothèque de lecture et d’écriture de raster multiformats. Il est utilisé comme premier mécanisme pour lire les données raster dans MapServer. Il est hébergé sur http://www.gdal.org/
GEOS
Geometry Engine Open Source est un port C/C++ de Java Topology Suite. Il est utilisé pour les opérations algébrique géométrique pour déterminer si un polygone est contenu dans un autre polygone ou déterminer l’intersection résultante de deux ou plusieurs polygones. MapServer peut utiliser GEOS pour des opérations algébriques.
GML

Geography Markup Language est un standard de l’OGC qui définit un modèle abstrait pour les objets géographiques.

Voir aussi

Serveur WFS

GPX
Le format GPS eXchange est un schéma XML pour décrire les données GPS. OGR peut être utilisé pour transformer et afficher ces données avec MapServer.
Map Scale
Un traité d’échelle cartographique peut être trouvé sur about.com.
Mapfile
Mapfile est le langage déclaratif que MapServer utilise pour définir les connections aux données, la sémiologie des cartes, les modèles, et les directives des serveurs. Son format est proche du XML et hiérarchique, avec des balises fermantes END, mais le format n’est pas du xml.
MapScript
MapScript est une alternative aux applications CGI de mapserv qui vous permet de programmer l’API objet de MapServer en plusieurs langages.
Mercator
Wikipedia propose un excellent article de la projection Mercator.
OGC

L’Open Geospatial Consortium est un organisme de standardisation dans le domaine du SIG. MapServer gère de nombreux standards de l’OGC.

Voir aussi

Serveur WMS and WMS Time and Client WMS and Serveur WFS and Client WFS and Serveur WCS and Map Context and SLD and Filter Encoding WFS and Serveur SOS

OGR

OGR est la partie accès aux données vecteur de la bibliothèque GDAL. Elle fournit un accès à une multitude de formats de données.

Voir aussi

OGR

OM

Observations and Measurements est un standard de l’OGC qui définit un modèle abstrait pour les capteurs de données.

Voir aussi

Serveur SOS

OpenLayers
OpenLayers est une bibliothèque JavaScript pour des applications web de cartographie.
Proj.4
Proj4 est une bibliothèque pour projeter des données spatiales. Elle est utilisé par MapServer et GDAL et beaucoup d’autres bibliothèques SIG Open Source.
Projection
Une projection cartographique est une transformation mathématique d’un volume d’une sphère (3D) en un plan 2D. À cause des lois de l’univers, chaque types de projection doit faire des compromis sur comment et quelles géométries sont transformées.
Raster
Un raster est une grille rectangulaire de pixels. Essentiellement une image. Les rasters sont gérés dans MapServer avec un type de couche définie à RASTER, et une variété de formats est géré incluant GeoTIFF, JPEG, et PNG.
Shapefile

Les shapefiles sont de simples fichiers SIG vecteur contenant des points, des lignes ou des surfaces. Le format a été créé et publié par ESRI et il est très largement géré dans le monde SIG. Il est effectivement le format natif et le plus performant pour MapServer.

Voir aussi

Wikipedia

SLD

SLD est un standard de l’OGC qui permet la personnalisation de la sémiologie de cartes finales.

Voir aussi

SLD

SOS

SOS est un standard de l’OGC qui propose une API pour gérer des capteurs déployés et récupérer des données de capteurs et d’observations.

Voir aussi

Serveur SOS

Spherical Mercator
Spherical Mercator Spherical Mercator est une terme utilisé pour décrire la PROJECTION utilisé par plusieurs éditeurs d’API commerciales.
SVG

Scalable Vector Graphics est un format XML que MapServer peut renvoyer. Il est fréquemment utilisé dans les périphériques mobiles et les navigateurs.

Voir aussi

SVG

SWF

Shockwave Flash format que MapServer peut générer en sortie.

Voir aussi

Sortie Flash

SWIG
Simplified Wrapper Interface Generator est la bibliothèque que MapServer utilise pour générer des liaisons de langages pour tous les langages autre que C/C++ et PHP. MapScript décrit ces liaisons.
Tileindex

Un tileindex est un Shapefile ou une autre source de données Vector qui contient l’empreinte des couvertures de données Raster. MapServer peut utiliser un tileindex pour afficher un répertoire de données raster. Le tileindex permet à MapServer de lire seulement les données qui intersectent l’étendue des cartes demandées, plutôt que de lire toutes les données.

Voir aussi

Index de tuiles

Vector
Objets géographiques décrits par des géométries (point, ligne, polygone) sur un plan cartésien (généralement).
WCS

WCS est un standard de l’OGC qui décrit comment produire systématiquement des données cartographiques Raster structurées à partir d’un service et les renvoyer à un client.

Voir aussi

Serveur WCS and Cas d’utilisations WCS

WFS

WFS est un standard de l’OGC qui décrit comment produire systèmatiquement des données cartographiques Vector structurées à partir d’un service et les renvoyer à un client.

Voir aussi

Serveur WFS and Client WFS

WMC

Web Map Context est un standard de l’OGC qui permet de partager des vues de cartes de couches WMS dans plusieurs applications.

Voir aussi

Map Context

WMS

WMS est un standard de l’OGC qui décrit comment produire systèmatiquement des images cartographiques à partir d’un service et de les renvoyer à un client.

Voir aussi

Serveur WMS and Client WMS
